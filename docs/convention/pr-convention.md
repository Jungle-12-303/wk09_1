# PR 작성 컨벤션

## PR 제목

커밋 메시지와 같은 형식을 사용한다.

```text
<type>: <한국어 설명>
```

예시:

```text
feat: 알람 클록 sleep list 기반 대기를 구현
fix: timer_sleep의 0 이하 tick 처리를 보정
docs: Pintos 원문 주석 번역본만 남겨
```

## PR 설명

PR 설명은 아래 구조를 기본으로 사용한다.

```markdown
## 무엇을 변경했나요?

- 변경 내용을 간단히 설명

## 왜 변경했나요?

- 변경 이유와 배경 설명

## 어떻게 확인했나요?

- 실행한 테스트나 확인 명령 작성

## 리뷰어가 보면 좋은 점

- 충돌 가능성, 주의할 파일, 의사결정이 필요한 부분 작성
```

## 작성 원칙

- 한국어로 작성한다.
- 변경 결과를 중심으로 쓴다.
- 테스트 결과는 명령어와 결과를 함께 적는다.
- 충돌 가능성이 있는 파일은 숨기지 않고 명시한다.
- 단순 변경이면 짧게 쓰고, 영향 범위가 넓으면 배경을 추가한다.

## 리뷰 요청 문구

리뷰 요청은 구체적으로 작성한다.

좋은 예:

```text
timer.c와 thread.c의 sleep/wakeup 흐름 위주로 봐주세요.
```

```text
주석 정리 PR이라 코드 동작 변경이 없는지 중심으로 봐주세요.
```

피할 예:

```text
리뷰 부탁드립니다.
```
